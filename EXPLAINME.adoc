== Claims Substantiation

=== Claim 1: "Solutions are verified using Z3 SMT solver, ensuring completed puzzles are mathematically valid"

**How it works:** The game constructs constraint systems describing each puzzle's solution space (e.g., "x + y = 10 AND x > y"). When a player submits a solution, `src/verification/` encodes the player's answer as additional constraints. The Z3 solver then checks satisfiability: if SAT, the solution is mathematically correct. The verification happens synchronously in `src/main.rs`'s game loop or asynchronously in the level editor (`src/ui/`), ensuring no invalid solution ever produces an achievement or leaderboard entry.

**Caveat:** Z3 verification is sound (correct solutions always verified) but depends on puzzle correctness. If a puzzle's constraint system is malformed, Z3 may return false negatives (reject correct solutions). Puzzles must be audited during creation in the level editor.

**Evidence:** `src/verification/` contains Z3 solver integration; `src/main.rs` invokes verification on solution submission; `src/game/puzzles.rs` defines puzzle constraint structure.

=== Claim 2: "Steam Integration - Achievements, leaderboards, cloud saves"

**How it works:** The Steam SDK (third-party C binding) is integrated in `src/steam/`. Game state (puzzle completion, solve times, high scores) is persisted to Steam Cloud via the `steam_cloud_save()` function. Achievements are unlocked through the `steam_unlock_achievement()` call when the solver completes a puzzle. Leaderboards are updated via `steam_submit_leaderboard_score()`, which ranks players by solution time. All Steam calls are wrapped in safe Rust APIs that translate between game state and Steam's data model.

**Caveat:** Steam is optional—the game runs offline without achievements or cloud saves. If Steam SDK is unavailable, fallback is local file-based persistence. Leaderboard cheating is possible if timestamps are manipulated client-side; production deployments should verify times server-side.

**Evidence:** `src/steam/` directory contains Steam integration; `src/main.rs` initializes Steam context; `src/game/` calls steam_* functions on state changes.
